| 维度 | P1（OWL） | P2（Prolog） | P3（Jena） | P4（模糊） | P5（贝叶斯） |

### 模糊 vs 概率：P4 vs P5 核心区别

```
病例：猫，发热 + 呕吐 + 腹泻

P4（模糊推理）：
  覆盖率=1.0, 强度=0.8, 排除度=0.0 → Mamdani 规则触发 → 置信度=0.86
  语义："症状与猫瘟的模糊匹配程度高"

P5（贝叶斯推理）：
  P(猫瘟)×P(发热|猫瘟)×P(呕吐|猫瘟)×P(腹泻|猫瘟)×P(¬咳嗽|猫瘟)×P(¬流鼻涕|猫瘟)
  → 归一化 → P(猫瘟|症状)=91.3%
  语义："给定这些症状，猫瘟的概率是91.3%"
```

**关键区别**：
- P4 的 0.86 = "匹配程度"（模糊规则输出），没有概率含义
- P5 的 91.3% = "后验概率"（贝叶斯定理），有严格的概率论保证
- P4 不利用先验知识（常见病和罕见病同等对待）
- P5 的先验概率使常见病天然占优（猫感冒 P=0.12 > 猫艾滋 P=0.01）
- P4 的排除症状"降低置信度"，P5 的排除症状通过 P(¬S|D) "降低似然"

## 技术原理

### 1. 朴素贝叶斯网络结构

```
    疾病D（根节点）
   / | \ \ \
  S₁ S₂ S₃ S₄ S₅（症状节点，叶子）
```

- 疾病是根节点，有先验概率 P(D)
- 症状是叶子节点，条件概率 P(S|D) 和 P(S|¬D)
- 朴素贝叶斯假设：给定疾病，症状条件独立

### 2. 条件概率表（CPT）

每个疾病维护一张 CPT，记录每个症状的：
- `present`: P(S|D) — 疾病存在时症状出现的概率（敏感度）
- `absent`: P(S|¬D) — 疾病不存在时症状出现的概率（假阳性率）

### 3. 贝叶斯推理

```
P(D|S₁,...,Sₙ) ∝ P(D) × ∏ P(Sᵢ|D) × ∏ P(¬Sⱼ|D)
```

- 先验 P(D)：疾病的base rate
- 似然 ∏ P(Sᵢ|D)：已出现症状的联合概率
- 似然 ∏ P(¬Sⱼ|D)：未出现症状的联合概率
- 归一化：除以所有疾病的似然之和

## 快速开始

```bash
cd P5/src

# 1. 构建知识库（首次运行）
python kb_builder.py

# 2. 交互式诊断
python diagnosis.py

# 3. 从 JSON 文件诊断
python diagnosis.py ../../shared_data/sample_case.json
```
