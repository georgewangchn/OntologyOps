## 一、引言：模糊推理的盲区

P1-P3 解决了确定性推理的问题：症状有/无 → 疾病是/否。P4 引入模糊推理，让症状有了严重度，输出连续置信度。但 P4 有一个盲区：

**P4 不知道疾病有多常见。**

猫感冒和猫艾滋病的症状可能都匹配得很好，但猫感冒很常见（发病率 12%），猫艾滋病很罕见（发病率 1%）。在没有额外信息时，理性的人应该更倾向于猫感冒。

P4 的模糊推理不区分这一点——它只看"症状匹配程度"，不看"疾病本身的概率"。

> **患者**：「医生，我的猫发热、呕吐、腹泻。」  
> **医生**：「这三个症状猫瘟和猫肠炎都有。但猫瘟的发病率是猫肠炎的两倍多，所以首先考虑猫瘟。」

医生在做诊断时，会不自觉地运用"先验概率"——常见病优先。P5 用贝叶斯推理将这个直觉形式化。

这正是《当 LLM 不够用了》第四章 4.4 节的核心论点：

> 概率推理不是"匹配程度"，而是"在给定证据下，疾病的概率是多少"——有严格的数学基础。

## 二、贝叶斯推理的核心思想

贝叶斯定理：

```
P(D|S) = P(S|D) × P(D) / P(S)
```

翻译成自然语言：

```
P(D)   → 先验概率——在看到任何症状之前，这只猫得猫瘟的概率是多少？
P(S|D) → 似然——如果这只猫真的得了猫瘟，出现发热呕吐腹泻的概率是多少？
P(D|S) → 后验概率——在看到发热呕吐腹泻之后，这只猫得猫瘟的概率是多少？
```

关键洞察：**先验概率让常见病天然占优**。猫感冒的先验（0.12）是猫艾滋病的 12 倍，所以同样的症状匹配程度下，猫感冒的后验概率更高。

### 与 P1-P4 的根本区别

```
维度            P1-P3（确定性）     P4（模糊）         P5（贝叶斯）
输出语义        "是/否"             "匹配度高/低"      "概率91.3%"
先验知识        不使用              不使用              核心输入
未出现症状      不影响              不影响              P(¬S|D) 降低后验
罕见病处理      与常见病同等         与常见病同等        先验天然压低
数学基础        描述逻辑/规则        模糊集合论          概率论（贝叶斯定理）
```

P4 的 0.86 = "症状与疾病的模糊匹配程度高"，没有概率含义。  
P5 的 91.3% = "给定这些症状，猫瘟的概率是91.3%"，有严格的概率论保证。

## 三、朴素贝叶斯网络结构

P5 采用最简单的贝叶斯网络结构——朴素贝叶斯：

```
    疾病D（根节点）
   / | \ \ \
  S₁ S₂ S₃ S₄ S₅（症状节点）
```

"朴素"假设：给定疾病，症状之间条件独立。这个假设在医学上不完全成立（发热和脱水显然相关），但实践中效果出奇地好——因为：

1. 计算大幅简化：不需要建模症状间的相关性
2. 即使独立性假设不成立，分类结果通常仍然正确
3. 需要的参数量少：每种疾病只需要先验概率 + 各症状的条件概率

### 条件概率表（CPT）

每个疾病维护一张条件概率表。以猫瘟为例：

```
症状      P(S|D) 疾病存在时    P(S|¬D) 疾病不存在时    似然比 LR
发热      0.90                 0.15                     6.0x
呕吐      0.85                 0.10                     8.5x
腹泻      0.80                 0.12                     6.7x
咳嗽      0.05                 0.20                     0.25x
流鼻涕    0.03                 0.18                     0.17x
```

三个核心概念：

```
P(S|D)                    → 敏感度——得了这个病时，出现这个症状的概率
P(S|¬D)                   → 假阳性率——没得这个病时，出现这个症状的概率
似然比 LR = P(S|D)/P(S|¬D) → 症状对疾病的证据强度
```

LR > 1 表示症状支持疾病，LR < 1 表示症状反对疾病。发热的 LR = 6.0x 意味着"发热"这个症状让猫瘟的可能性提升了 6 倍。

### 先验概率

```
疾病          先验 P(D)    含义
猫感冒        0.12         最常见
犬感冒        0.12         最常见
猫瘟          0.08         常见
犬细小病毒    0.07         常见
猫肠炎        0.06         较常见
犬冠状病毒    0.05         较常见
猫尿路感染    0.04         一般
犬尿路感染    0.04         一般
犬副流感      0.03         少见
猫艾滋病      0.01         罕见
```

先验概率从哪来？来自流行病学统计——某地区宠物医院一段时间内的就诊数据。在没有症状信息时，先验概率就是我们对疾病的最佳估计。

## 四、推理流程

### 4.1 计算后验

对每个疾病 D，计算：

```
P(D|S₁,...,Sₙ) ∝ P(D) × ∏ P(Sᵢ|D) × ∏ P(¬Sⱼ|D)

P(D)             → 先验概率
∏ P(Sᵢ|D)       → 已出现症状的联合似然
∏ P(¬Sⱼ|D)      → 未出现症状的联合似然（= ∏ (1 - P(Sⱼ|D))）
```

**关键点：未出现的症状也参与推理！** 这是 P5 与 P1-P4 的核心区别。

在 P1-P3 中，猫瘟的必要症状是发热、呕吐、腹泻。如果用户只报告了发热和呕吐，P1-P3 不会因为"没报告腹泻"而降低置信度——腹泻"不存在"只是不贡献证据。

在 P5 中，腹泻未出现意味着 P(¬腹泻|猫瘟) = 1 - 0.80 = 0.20 参与计算。这个 0.20 会大幅拉低猫瘟的后验——因为猫瘟患者 80% 会有腹泻，没有腹泻是一个反对证据。

### 4.2 归一化

将所有疾病的后验值归一化为概率分布：

```
P(Dₖ|S) = posterior(Dₖ) / Σ posterior(Dᵢ)
```

归一化保证所有疾病的后验概率之和为 1。这不是任意操作——它是概率论的要求，确保结果是一个合法的概率分布。

### 4.3 实际推理结果

```
病例：猫，发热 + 呕吐 + 腹泻

P5 贝叶斯推理结果：
  1. 猫瘟      P(D|S) = 91.32%  █████████ [高概率]
  2. 猫肠炎     P(D|S) =  8.31%  [低概率]
  3. 猫艾滋病    P(D|S) =  0.34%  [低概率]
  4. 猫感冒     P(D|S) =  0.02%  [低概率]

对比 P4 模糊推理结果：
  1. 猫瘟      置信度 = 0.86  [高]
  2. 猫肠炎     置信度 = 0.50  [中]  ← P4 不会降到这么低
```

关键差异：P5 中猫肠炎只有 8.31%，因为猫肠炎的排除症状"发热"出现了，P(发热|猫肠炎)=0.10 很低，大幅拉低了后验。而 P4 中猫肠炎仍有 0.50 的置信度。

另一个差异：猫感冒的先验概率最高（0.12），但在 P5 中只有 0.02%。因为猫感冒的必要症状（打喷嚏、流鼻涕）一个都没出现，P(¬打喷嚏|猫感冒) × P(¬流鼻涕|猫感冒) = 0.30 × 0.18 = 0.054，极大地降低了后验。P4 中虽然猫感冒的覆盖率也很低，但没有先验的杠杆效应。

## 五、P4 vs P5：模糊 vs 概率

```
维度              P4（模糊推理）              P5（贝叶斯推理）
输出语义          "匹配程度"                  "后验概率"
先验知识          不使用                      核心输入
未出现症状        不影响（只看已出现的）       通过 P(¬S|D) 降低后验
排除症状          降低置信度                  通过低 P(S|D) 降低似然
罕见病处理        与常见病同等                先验概率天然压低
数学基础          模糊集合论                  概率论（贝叶斯定理）
0.86 的含义       "症状与疾病模糊匹配度高"    无此输出
91.3% 的含义      无此输出                    "给定症状，疾病概率91.3%"
```

**一句话总结**：P4 回答"症状和疾病有多匹配？"，P5 回答"给定症状，疾病的概率是多少？"。

### 什么时候用 P4，什么时候用 P5？

**用 P4**：当没有可靠的先验概率数据时。模糊推理不需要发病率统计，只需要专家定义隶属度函数和规则。

**用 P5**：当有可靠的先验概率和条件概率数据时。贝叶斯推理的输出更有决策价值——"91.3%"比"0.86"更容易被医生和患者理解。

在实际系统中，P4 和 P5 不是互斥的——P6 会将两者融合。

## 六、推理链可解释性

P5 的解释比 P4 更有数学根基：

```
[猫瘟]
  先验 P(D) = 8.00% → 后验 P(D|S) = 91.32%
  似然比 = 11.41x  ← 先验被放大了11倍

  症状贡献：
    发热:    P(S|D)=0.90 ✅ [支持]      ← 强证据
    呕吐:    P(S|D)=0.85 ✅ [支持]      ← 强证据
    腹泻:    P(S|D)=0.80 ✅ [支持]      ← 强证据
    咳嗽:    P(S|D)=0.05 ❌ [不支持]    ← 未出现，P(¬S|D)=0.95
    流鼻涕:  P(S|D)=0.03 ❌ [不支持]    ← 未出现，P(¬S|D)=0.97
```

每个症状的贡献可以用似然比 P(S|D)/P(S|¬D) 量化：
- 发热：0.90/0.15 = 6x（强支持猫瘟）
- 咳嗽未出现：P(¬S|D)/P(¬S|¬D) = 0.95/0.80 = 1.19x（轻微支持猫瘟，因为猫瘟不太咳嗽）

这种解释方式与临床医学的诊断推理完全一致——医生也在用"这个症状支持什么病、排除什么病"的思路。

### 与 P1-P4 的解释对比

```
系统    解释方式                                    量化程度
P1      "equivalent_to 匹配 → 推断为 D001"           无量化（是/否）
P2      "必要症状全匹配 → 确诊"                      置信度 = 匹配数/总数
P3      "前向链规则触发 → suspected"                 同 P2
P4      "覆盖率=高, 强度=高, 排除度=无 → 置信度=高"   模糊等级（高/中/低）
P5      "先验8% → 似然比11.4x → 后验91.3%"           精确概率 + 似然比
```

P5 的解释是唯一有严格数学基础的——每个数字都有概率含义。

## 七、知识库构建

### 先验概率的来源

P5 的 `kb_builder.py` 中硬编码了 10 种疾病的先验概率。这些值来自宠物医疗的常见经验估计：

```python
DISEASE_PRIORS = {
    "D001": 0.08,   # 猫瘟
    "D002": 0.12,   # 猫感冒（最常见）
    "D003": 0.06,   # 猫肠炎
    "D004": 0.07,   # 犬细小病毒
    "D005": 0.12,   # 犬感冒（最常见）
    "D006": 0.05,   # 犬冠状病毒
    "D007": 0.04,   # 猫尿路感染
    "D008": 0.04,   # 犬尿路感染
    "D009": 0.01,   # 猫艾滋病（罕见）
    "D010": 0.03,   # 犬副流感
}
```

在生产系统中，这些值应该从电子病历数据库统计得到。

### 条件概率表的设计

每种疾病 3-5 个症状的 CPT 也是专家设定的。设计原则：

```
症状类型          P(S|D) 设定      说明
必要症状          0.80-0.90        高概率但非100%，存在非典型病例
排除症状          0.03-0.10        低概率，出现时强烈反对该疾病
无关症状          ≈ P(S|¬D)        LR ≈ 1，不提供信息
P(S|¬D) 基线      0.10-0.20        该症状在一般群体中的出现率
```

### 与 diseases.csv 的关系

P5 的知识源仍然是 `shared_data/diseases.csv`，但做了"范式翻译"：

```
CSV 语义     CPT 翻译               说明
必要症状     P(S|D) = 0.80-0.90     高概率，但不是确定性约束
排除症状     P(S|D) = 0.03-0.10     低概率，但不是硬排除
充分症状     暂未建模               未来可扩展为确诊触发器
```

这种翻译是必要的——贝叶斯框架中没有"确定性必要"和"硬排除"的概念，一切都是概率。

## 九、总结

P5 贝叶斯推理与 P4 模糊推理的关键区别不在于"连续 vs 离散"——两者都输出连续值。区别在于：

1. **概率语义**：P5 的输出是严格的后验概率，可以回答"有多大概率"
2. **先验利用**：P5 利用疾病发病率作为先验，常见病天然占优
3. **负证据**：P5 中未出现的症状也是证据（P(¬S|D)），P4 忽略未出现的症状
4. **可解释性**：P5 的似然比有明确的概率含义，P4 的模糊规则是启发式的

P1-P5 五种范式的递进路线：

```
阶段      范式          核心能力
P1-P3     确定性推理     不同知识表示，同一确定性本质
P4        模糊推理       处理"边缘模糊"，但不处理概率
P5        概率推理       处理"概率不确定性"，有严格数学基础
```

> 「确定性推理负责'法律的条文'，不确定性推理负责'法官的自由裁量'，概率推理负责'陪审团的裁定'。」
