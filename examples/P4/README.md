> **多范式推理实战营 — 项目 4/6**
> 推理范式：**模糊推理（Mamdani 模糊逻辑 + 隶属度函数 + 去模糊化）**
> 对比 P1：**OWL 本体推理（HermiT + SWRL）**
> 对比 P2：**Prolog 规则推理（SWI-Prolog + SLD 归结）**
> 对比 P3：**Jena 前向链规则推理 + SPARQL 查询**

> 📖 同一领域问题（宠物疾病诊断），第四种推理范式。P1-P3 都是确定性推理（症状有/无 → 疾病是/否），P4 引入不确定性推理（症状严重度 → 疾病置信度），展示模糊逻辑如何处理"边缘模糊"的诊断问题。

## P1-P4 四种推理范式对比

| 维度 | P1（OWL + HermiT） | P2（Prolog） | P3（Jena Fuseki） | P4（模糊推理） |
|------|-------------------|-------------|-------------------|---------------|
| 逻辑基础 | 描述逻辑（DL） | Horn 子句逻辑 | 前向链规则 + RDF | **模糊逻辑（Mamdani）** |
| 世界假设 | OWA（开放世界） | CWA（封闭世界） | CWA（negation-as-failure） | **模糊（隶属度 0-1）** |
| 输入类型 | 症状有/无（二元） | 症状有/无（二元） | 症状有/无（二元） | **症状严重度（连续 0-1）** |
| 输出类型 | 确诊/排除（二元） | 确诊/疑似/排除 | 确诊/疑似/排除 | **置信度（连续 0-1）** |
| 排除语义 | 命中排除 → 完全排除 | `\+` → 完全排除 | `noValue` → 完全排除 | **命中排除 → 降低置信度** |
| 推理引擎 | HermiT（Java，JVM） | SWI-Prolog（C） | Jena（Java，JVM） | **scikit-fuzzy（纯 Python）** |
| 知识来源 | 领域专家建模 | 领域专家写规则 | 领域专家写规则 | **领域专家定义隶属度函数** |
| 外部依赖 | JVM + owlready2 补丁 | SWI-Prolog | Docker + Jena Fuseki | **无（纯 Python）** |

### 确定性 vs 不确定性：核心区别

```
病例：猫，体温 39.5°C，呕吐多次，水样暗红腹泻

P1-P3（确定性推理）：
  症状是二元的 → 发热✓ 呕吐✓ 腹泻✓ → 猫瘟（确诊，置信度=1.0）
  排除是二元的 → 发热命中猫肠炎排除症状 → 猫肠炎完全排除（不出现在结果中）

P4（模糊推理）：
  症状有严重度 → 发热(0.8) 呕吐(0.7) 腹泻(0.9) → 猫瘟（置信度=0.86）
  排除有程度   → 发热(0.8) 命中猫肠炎排除 → 猫肠炎置信度降低至 0.50（仍出现在结果中）
```

**关键区别**：P1-P3 中猫肠炎被完全排除（不在结果中），P4 中猫肠炎仍出现但置信度降低。模糊推理不会"一刀切"排除，而是根据排除症状的严重度**降低**置信度。

### 为什么需要模糊推理？

引用《当LLM不够用了》第四章 4.4 节：

> 确定性推理保证结论的可解释性和一致性。不确定性推理保证覆盖现实世界的灰度地带。
> 
> 确定性推理负责"法律的条文"（硬规则），不确定性推理负责"法官的自由裁量"（模糊信号）。

宠物医疗场景中，"发热"不是有或无的问题——39.5°C 是高烧，38.8°C 是低烧，两者的临床意义完全不同。P1-P3 把它们都当作"有发热"，丢失了严重程度信息。P4 用隶属度函数保留了这一信息。

## 技术原理

### 1. 双输入模糊化（Fuzzification）

P4 采用**覆盖率 + 强度**双输入设计，将"有多少必要症状出现"和"已出现的症状有多严重"分离为两个正交维度：

**覆盖率**（必要症状的出现比例，0-1）：

```
覆盖率 = 已出现必要症状数 / 必要症状总数

例：猫瘟 necessary=[发热, 呕吐, 腹泻]，病例有 3 个全在
  覆盖率 = 3/3 = 1.0

例：猫艾滋 necessary=[发热, 淋巴结肿大]，病例只有发热
  覆盖率 = 1/2 = 0.5

隶属度函数（梯形）：
  低 [0, 0, 0.15, 0.35]    ─── ╲
  中 [0.25, 0.45, 0.55, 0.75]    ╱─╲
  高 [0.65, 0.85, 1.0, 1.0]         ╱───
```

**强度**（已出现症状的严重度均值，0-1）：

```
强度 = 已出现必要症状的严重度均值（缺失症状不计入）

例：猫瘟 3 个全在，严重度 0.8/0.7/0.9
  强度 = (0.8 + 0.7 + 0.9) / 3 = 0.80

例：猫艾滋只有发热(0.8)
  强度 = 0.8 / 1 = 0.80（只算已出现的，不因缺失症状拉低）

隶属度函数同覆盖率
```

**排除度**（排除症状的出现程度，0-1）：

```
排除度 = 排除症状严重度的最大值（模糊 OR）

例：猫肠炎 exclusion=[发热, 咳嗽]
  发热严重度=0.8, 咳嗽严重度=0.0（未出现）
  排除度 = max(0.8, 0.0) = 0.80

隶属度函数（梯形）：
  无 [0, 0, 0.05, 0.15]   ─── ╲
  有 [0.1, 0.3, 1.0, 1.0]      ╱───
```

> 📝 **为什么用双输入而非单输入？**
> 
> 旧设计的"匹配度"= 覆盖率×强度混合，无法区分两种医学语义完全不同的情况：
> - "3/3 症状全在但都是低烧" → 匹配度低（不合理，症状都在）
> - "1/3 症状在但极高烧" → 匹配度低（合理，缺核心症状）
> 
> 双输入设计分离后：
> - 3/3 全在低烧 → 覆盖率=高, 强度=低 → 中置信度（症状都在，只是轻）
> - 1/3 在极高烧 → 覆盖率=低, 强度=高 → 低置信度（缺核心症状）

### 2. 模糊规则（Mamdani IF-THEN，三输入 12 条）

12 条规则覆盖覆盖率×强度×排除度的所有组合：

**规则矩阵（排除度=无时）：**

| 覆盖率\强度 | 低 | 中 | 高 |
|------------|-----|-----|-----|
| 低 | 低 | 低 | 低 |
| 中 | 低 | 中 | 中 |
| 高 | 中 | 中 | **高** |

**规则矩阵（排除度=有时）：**

| 覆盖率\强度 | 低 | 中 | 高 |
|------------|-----|-----|-----|
| 低 | 低 | 低 | 低 |
| 中 | 低 | 低 | 低 |
| 高 | 低 | 低 | **中** |

```python
# 核心确诊规则：全覆盖 + 高强度 + 无排除 → 高置信度
IF 覆盖率=高 AND 强度=高 AND 排除度=无 THEN 置信度=高

# 排除降级规则：全覆盖 + 高强度 + 有排除 → 中置信度（降一级，非完全排除）
IF 覆盖率=高 AND 强度=高 AND 排除度=有 THEN 置信度=中

# 缺核心症状：覆盖率低 → 低置信度（不管多严重）
IF 覆盖率=低 THEN 置信度=低

# ...共 12 条规则
```

对比 P1-P3 的规则：
- P1 SWRL：`has(?p,?s) ∧ necessary(?d,?s) → suspected(?p,?d)` — 二元触发
- P2 Prolog：`forall(necessary(D,S), has(P,S))` — 全匹配才确诊
- P3 Jena：`noValue(?case :excluded ?disease) → diagnosed` — 无排除才确诊
- **P4 模糊**：`覆盖率=高 AND 强度=高 AND 排除度=无 → 置信度=高` — 隶属度触发，连续输出

### 3. 模糊推理（Mamdani Inference）

每条规则的激活强度 = min(条件隶属度)（模糊 AND）。

以猫瘟为例（覆盖率=1.0, 强度=0.80, 排除度=0.0）：

```
规则11：覆盖率=高(μ=1.0) AND 强度=高(μ=0.75) AND 排除度=无(μ=1.0) → 激活强度=0.75 → 置信度=高
其余规则：不触发或激活强度=0

合成结果：置信度=高，μ=0.75
```

以猫肠炎为例（覆盖率=1.0, 强度=0.80, 排除度=0.80）：

```
规则12：覆盖率=高(μ=1.0) AND 强度=高(μ=0.75) AND 排除度=有(μ=1.0) → 激活强度=0.75 → 置信度=中
其余规则：不触发或激活强度=0

合成结果：置信度=中，μ=0.75
```

### 4. 去模糊化（Defuzzification）

将模糊输出转为连续值，使用重心法（centroid）：

```
置信度输出模糊集合 → 重心法 → 连续置信度值

猫瘟：置信度=高(μ=0.75) → 重心 ≈ 0.86
猫肠炎：置信度=中(μ=0.75) → 重心 ≈ 0.50
猫艾滋：置信度=低(μ=0.75) → 重心 ≈ 0.14
```

对比 P1-P3 的输出：
- P1：`confidence = match_count / total_count + 0.1`（离散比率 + 经验偏移）
- P2：`Confidence is M / N`（Prolog 算术，离散比率）
- P3：`confidence = matched_count / nec_count`（SPARQL 计数，离散比率）
- **P4**：重心法去模糊化（连续值，由隶属度函数形状决定）

## 推理链详解

以 sample_case（猫，发热 39.5°C + 呕吐多次 + 水样暗红腹泻）为例：

```
① 症状严重度计算（模糊化输入）
   发热: 39.5°C → 0.80（高烧范围）
   呕吐: 多次   → 0.70（中高频）
   腹泻: 水样+暗红 → 0.90（严重，含血色）

② 覆盖率 + 强度 + 排除度计算
   猫瘟   necessary=[发热,呕吐,腹泻] → 覆盖率=3/3=1.0  强度=(0.8+0.7+0.9)/3=0.80
           exclusion=[咳嗽,流鼻涕]  → 排除度=max(0,0)=0.00

   猫肠炎 necessary=[腹泻,呕吐]      → 覆盖率=2/2=1.0  强度=(0.9+0.7)/2=0.80
           exclusion=[发热,咳嗽]    → 排除度=max(0.8,0)=0.80

   猫艾滋 necessary=[发热,淋巴结肿大] → 覆盖率=1/2=0.5  强度=0.8/1=0.80
           exclusion=[腹泻]        → 排除度=max(0.9)=0.90

③ Mamdani 模糊推理（规则触发）
   猫瘟:   覆盖率=高 ∧ 强度=高 ∧ 排除度=无 → 规则11触发 → 置信度=高
   猫肠炎: 覆盖率=高 ∧ 强度=高 ∧ 排除度=有 → 规则12触发 → 置信度=中
   猫艾滋: 覆盖率=中 ∧ 强度=高 ∧ 排除度=有 → 规则6触发  → 置信度=低

④ 去模糊化（重心法）
   猫瘟:   置信度=高 → 0.86
   猫肠炎: 置信度=中 → 0.50
   猫艾滋: 置信度=低 → 0.14

⑤ 最终结论
   → 猫瘟（0.86 [高]）> 猫肠炎（0.50 [中]）> 猫艾滋病（0.14 [低]）
```

### 与 P1-P3 推理链的关键差异

| 疾病 | P1（OWL） | P2（Prolog） | P3（Jena） | P4（模糊） |
|------|----------|-------------|-----------|-----------|
| 猫瘟 | 确诊（0.99） | 确诊（1.00） | 确诊（1.00） | **高（0.86）** |
| 犬细小 | 疑似（0.77） | 疑似（0.67） | 确诊（1.00） | 不出现（物种过滤） |
| 猫肠炎 | **排除** | **排除** | **排除** | **中（0.50）** |

**猫肠炎是关键区别**：P1-P3 中因排除症状"发热"命中 → 完全排除（不在结果中）；P4 中因排除症状"发热"严重度高 → 置信度降为中等（仍在结果中）。双输入设计下，猫肠炎的覆盖率=高+强度=高（必要症状全在且严重），但排除度=有（发热命中），所以降一级为"中"而非完全压低为"低"——匹配证据强时排除只降一级，匹配证据弱时排除直接压低。

### reasoner.py（Mamdani 模糊推理）

```python
import numpy as np
import skfuzzy as fuzz
from skfuzzy import control as ctrl

# 构建模糊控制器（3 个前件 + 1 个后件）
coverage_var = ctrl.Antecedent(np.arange(0, 1.01, 0.01), '覆盖率')
intensity_var = ctrl.Antecedent(np.arange(0, 1.01, 0.01), '强度')
exclude_var = ctrl.Antecedent(np.arange(0, 1.01, 0.01), '排除度')
confidence_var = ctrl.Consequent(np.arange(0, 1.01, 0.01), '置信度')

# 设置隶属度函数
coverage_var['高'] = fuzz.trapmf(coverage_var.universe, [0.65, 0.85, 1.0, 1.0])

# 定义规则（三输入）
rule11 = ctrl.Rule(
    coverage_var['高'] & intensity_var['高'] & exclude_var['无'],
    confidence_var['高']
)

# 构建控制器
controller = ctrl.ControlSystem([rule1, rule2, ..., rule12])

# 执行推理
sim = ctrl.ControlSystemSimulation(controller)
sim.input['覆盖率'] = 1.0
sim.input['强度'] = 0.80
sim.input['排除度'] = 0.00
sim.compute()
confidence = sim.output['置信度']  # → 0.86
```

### utils.py（覆盖率 + 强度计算）

```python
def compute_coverage(disease_necessary, case):
    """覆盖率 = 已出现症状数 / 必要症状总数（与 P1-P3 匹配率同语义）"""
    case_symptoms = case.get("symptoms", [])
    matched = [s for s in disease_necessary if s in case_symptoms]
    return len(matched) / len(disease_necessary)

def compute_intensity(disease_necessary, case, baselines):
    """强度 = 已出现症状的严重度均值（仅统计已出现的，缺失不计入）"""
    case_symptoms = case.get("symptoms", [])
    matched = [s for s in disease_necessary if s in case_symptoms]
    if not matched:
        return 0.0
    severities = [compute_symptom_severity(s, case, baselines) for s in matched]
    return sum(severities) / len(severities)
```

## 与朴素贝叶斯的对比

朴素贝叶斯是另一种处理不确定性的方法，但它与模糊推理有本质区别：

| 维度 | 模糊推理（P4） | 朴素贝叶斯 |
|------|--------------|-----------|
| **知识来源** | 领域专家定义隶属度函数 | 从数据统计学习概率 |
| **推理性质** | 演绎（规则驱动） | 归纳（数据驱动） |
| **可解释性** | 每条规则可追溯 | 概率黑盒 |
| **与本体关系** | 可扩展为 Fuzzy OWL，嵌入本体 | 独立范式，不嵌入本体 |
| **数据需求** | 低（不需要训练集） | 高（需要大量标注病例） |
| **书中定位** | 4.4 不确定性推理 | 4.6 ML 方法（与推理对比） |

引用第四章 4.6 节：

> 模糊推理的隶属度函数需要领域专家定义，不是纯数据驱动——这恰好防止了"统计分数伪装成逻辑结论"的问题。
