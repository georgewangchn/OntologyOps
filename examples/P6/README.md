# P6 · 多范式贝叶斯元推理系统

> **多范式推理实战营 — 项目 6/7**
> 推理范式：**贝叶斯元推理（似然比融合）**
> P2(Prolog) + P4(模糊) + P5(贝叶斯) 并行推理 → 似然比转换 → 贝叶斯乘法融合 → 归一化

## 架构

```
┌─────────────────────────────────────────────────┐
│           病例输入 (症状 + 严重度)                 │
├────────────┬────────────────┬───────────────────┤
│  P2 Prolog │   P4 Mamdani   │  P5 Naive Bayes   │
│  确定性推理  │   模糊推理      │  贝叶斯推理        │
│  CWA       │   连续严重度    │  先验 + CPT        │
│            │                │                   │
│  增量:规则  │  增量:严重度    │  增量:先验+CPT     │
├────────────┼────────────────┼───────────────────┤
│  -> LR_struct │ -> LR_fuzzy  │ -> LR_bayesian   │
│  确诊=5.0    │ exp(k*(c-0.5))│ posterior/prior  │
│  疑似=1.5    │               │                  │
│  排除=0.1    │               │                  │
├────────────┴────────────────┴───────────────────┤
│     Bayesian Meta-Reasoner (贝叶斯元推理器)        │
│  P_final(D) ∝ P_prior(D) × LR_struct × LR_fuzzy │
│              × LR_bayesian                       │
│  归一化 → 最终后验概率分布                         │
└─────────────────────────────────────────────────┘
```

## 为什么不用旧版分层仲裁

| 问题 | 旧版分层仲裁 | 新版贝叶斯元推理 |
|------|------------|----------------|
| P1/P2/P3 知识源冗余 | 三引擎投票=一人投三票 | 只用 P2，消除冗余 |
| P4 与 P1-P3 输入重叠 | coverage 逻辑相同 | 各引擎增量正交 |
| 权重 0.6/0.4 无理论依据 | 加权平均语义不匹配 | 似然比乘法有贝叶斯定理保证 |

## 似然比映射

| 引擎 | 输出 | LR 映射 | 含义 |
|------|------|---------|------|
| P2 确诊 | confidence=1.0 | LR=5.0 | 强证据支持 |
| P2 疑似 | confidence≥0.5 | LR=1.5 | 弱证据支持 |
| P2 排除 | confidence=0.0 | LR=0.1 | 强证据反对 |
| P4 高匹配 | confidence≈0.8 | LR≈4.0 | 症状严重度高→支持 |
| P4 低匹配 | confidence≈0.2 | LR≈0.4 | 症状严重度低→反对 |
| P5 后验 | posterior | LR=posterior/prior | 后验相对先验提升 |
