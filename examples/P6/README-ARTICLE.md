## 一、为什么需要融合

P1-P5 各自独立推理，各有优缺点：

| 引擎 | 优点 | 缺点 |
|------|------|------|
| P1 OWL | 双向推理，公理保证 | 不处理概率 |
| P2 Prolog | 递归推理，CWA 直接 | 过度排除 |
| P3 SPARQL | 前向链预计算 | 需要外部服务 |
| P4 模糊 | 处理灰度 | 无概率语义 |
| P5 贝叶斯 | 概率论保证 | 依赖先验质量 |

单一引擎的缺点，可以通过融合其他引擎来弥补。问题是：**当引擎们意见不一致时，谁的证据该被采信？**

> **场景**：一只猫发热 + 呕吐 + 腹泻。  
> P2（Prolog）说：确诊猫瘟（必要症状全匹配）。  
> P4（模糊）说：猫瘟置信度 0.86（高匹配），但猫肠炎也有 0.50（中匹配）。  
> P5（贝叶斯）说：猫瘟后验 91.3%，猫肠炎只有 8.3%。  
> **该信谁？猫肠炎到底要不要考虑？**

## 二、旧版方案的三个问题

最初的实现采用"分层仲裁"模式：P1+P2+P3 投票 → P4 模糊量化 → P5 概率校准 → 仲裁器加权(0.6/0.4)。从第一性原理审视后发现三个结构性缺陷：

### 问题 1：P1/P2/P3 知识源冗余

P1（OWL）、P2（Prolog）、P3（SPARQL）的知识源本质相同——同一份疾病-症状关系表的三种编码形式。三个引擎的置信度计算逻辑也完全相同（匹配数/必要症状数）。把它们放在"第一层"做投票，等价于一个人投三票，"一致性"几乎是必然的。

### 问题 2：P4 与 P1-P3 输入空间重叠

P4 模糊推理的三个输入中，`coverage`（覆盖率）的计算逻辑与 P1-P3 的置信度完全相同。P4 相对于 P1-P3 的真正增量仅为"症状严重度"（连续值），但旧版没有利用这个增量关系。

### 问题 3：权重 0.6/0.4 无理论依据

贝叶斯后验概率（P5）和模糊匹配度（P4）的语义不同。P5 的 0.8 = "后验概率 80%"，P4 的 0.8 = "模糊匹配度高"。把两者加权平均，就像把温度和湿度按 6:4 混合——单位不匹配。

## 三、贝叶斯元推理架构

```
病例输入 → ┬→ P2(Prolog) 确定性推理 → LR_struct
           ├→ P4(Mamdani) 模糊推理   → LR_fuzzy
           └→ P5(Naive Bayes) 概率推理 → LR_bayesian
                                            │
           Bayesian Meta-Reasoner ←────────┘
           P_final(D) ∝ P_prior(D) × LR_struct × LR_fuzzy × LR_bayesian
           归一化 → 最终后验概率分布
```

### 引擎选择：P1-P3 取其一

P1/P2/P3 共享同一知识源，只保留 P2（Prolog）。选择 P2 的理由：
- CWA（封闭世界假设）最适合诊断场景：未断言的症状 = 没有 → 可以直接排除
- P1 的 OWA 过于保守，P3 需要外部 Fuseki 服务

### 各引擎的增量信息

| 引擎 | 增量信息 | 语义 |
|------|---------|------|
| P2 | 疾病-症状必要性关系 + 排除关系 | 确定性：是/否 |
| P4 | 症状严重度（连续） | 模糊匹配度 |
| P5 | 先验概率 + 条件概率表 | 后验概率 |

三个引擎的增量信息是**正交的**——各自提供其他引擎没有的信息维度。这是融合有价值的前提。

## 四、似然比转换

似然比（Likelihood Ratio, LR）是医学诊断中的标准工具。LR > 1 表示证据支持疾病，LR < 1 表示证据反对疾病。

### 为什么选择似然比？

似然比有一个绝好的性质：**它可以跨范式对齐**。

- P2 输出的是离散判定（确诊/疑似/排除）
- P4 输出的是模糊匹配度（0-1 连续值）
- P5 输出的是后验概率（0-1 概率）

这三个值的语义完全不同，不能直接比较或加权。但它们都可以转换为"证据对疾病的支持程度"——即似然比。转换后，乘法融合就有了统一的"货币"。

### P2 确定性推理 → LR_struct

| P2 输出 | LR | 含义 |
|---------|-----|------|
| 确诊 | 5.0 | 强证据支持 |
| 疑似 | 1.5 | 弱证据支持 |
| 排除 | 0.1 | 强证据反对 |
| 未出现 | 1.0 | 无信息 |

P2 的 LR 是离散映射——确定性推理没有连续输出，只能按判定等级映射。这些值是专家设定的，方向正确（确诊支持、排除反对），量级合理（5.0 是强证据但不至于一锤定音）。

### P4 模糊推理 → LR_fuzzy

使用指数映射：`LR = exp(k × (confidence - 0.5))`

- confidence = 0.5 → LR = 1.0（中性，不改变后验）
- confidence = 1.0 → LR ≈ 4.48（强支持）
- confidence = 0.0 → LR ≈ 0.22（强反对）

指数映射保证 LR 始终为正，且支持/反对区间对称。参数 k=3.0 控制映射的灵敏度——k 越大，模糊匹配度对最终结果的影响越大。

### P5 贝叶斯推理 → LR_bayesian

`LR = posterior / prior`（后验相对先验的提升倍数）

- LR = 1.0 → 后验 = 先验（症状没提供额外信息）
- LR > 1.0 → 症状支持该疾病
- LR < 1.0 → 症状反对该疾病

P5 的 LR 是最自然的——它就是贝叶斯因子（Bayes Factor），有严格的概率论定义。P5 已经计算了先验和后验，LR 只是两者的比值。

## 五、贝叶斯乘法融合

```
P_final(D) ∝ P_prior(D) × LR_struct × LR_fuzzy × LR_bayesian
```

归一化后得到最终后验概率分布。这个公式有**贝叶斯定理的链式法则**保证——不是经验权重，而是概率论推导。

### 数学推导

贝叶斯定理的扩展形式：

```
P(D|E₁, E₂, E₃) ∝ P(D) × P(E₁|D)/P(E₁) × P(E₂|D)/P(E₂) × P(E₃|D)/P(E₃)
```

其中每个 `P(Eᵢ|D)/P(Eᵢ)` 就是似然比。如果三个证据在给定 D 的条件下独立，这个公式是精确的。

三个引擎的输出转为 LR 后，正好对应三个证据源。P2 提供结构性证据（症状-疾病规则匹配），P4 提供模糊证据（严重度匹配度），P5 提供概率证据（先验 + CPT）。它们的增量信息正交，近似满足条件独立假设。

### 冲突检测

如果各引擎的 LR 方向不一致（一个大于 1 支持，一个小于 1 反对），且最大 LR / 最小 LR 超过 5.0，标记冲突。

```
冲突示例：
  P2: LR = 5.0（确诊，强支持）
  P5: LR = 0.3（后验低于先验，反对）
  比值 = 5.0 / 0.3 = 16.7 > 5.0 → 标记冲突
```

冲突标记不改变推理结果，但会在输出中提醒用户：各引擎意见不一致，需人工复核。

## 六、推理结果示例

```
病例：猫，发热 + 呕吐 + 腹泻

1. 猫瘟    91.30%  █████████ [高概率]
   融合：P_prior(0.0500) × LR_P2(5.0) × LR_P4(3.2) × LR_P5(18.2)
   P2: conf=1.00 [确诊] LR=5.00
   P4: conf=0.86 [高]   LR=3.20
   P5: conf=0.91 [高概率] LR=18.20

2. 猫肠炎   5.20%  [低概率]
   融合：P_prior(0.0300) × LR_P2(0.1) × LR_P4(1.8) × LR_P5(0.8)
   P2: conf=0.00 [排除] LR=0.10
   P4: conf=0.62 [中]   LR=1.80
   P5: conf=0.02 [低概率] LR=0.80
```

### 结果解读

猫瘟：三个引擎一致支持——P2 确诊（LR=5.0），P4 高匹配（LR=3.2），P5 高后验（LR=18.2）。乘法融合后，先验 5% 被放大到 91.3%。

猫肠炎：P2 排除（LR=0.1），但 P4 模糊匹配仍有中等置信度（LR=1.8），P5 后验很低（LR=0.8）。三个引擎中两个反对、一个弱支持，融合后只有 5.2%。

关键观察：P4 对猫肠炎给出了 LR=1.8（弱支持），但 P2 的 LR=0.1（强反对）在乘法中起决定性作用——0.1 × 1.8 = 0.18，直接把猫肠炎压到低概率。这就是乘法融合的优势：**一个强反对证据可以否决多个弱支持证据**，与临床诊断的决策逻辑一致。

## 七、P1-P6 完整体系回顾

```
P1 (OWL/HermiT)     确定性 | OWA      知识编码: OWL本体 + SWRL规则
P2 (Prolog/SLD)     确定性 | CWA      知识编码: Horn子句 + 递归
P3 (Jena/SPARQL)    确定性 | 前向链    知识编码: RDF三元组
P4 (Mamdani 模糊)   不确定性 | 匹配度   知识编码: 隶属度函数 + 模糊规则
P5 (朴素贝叶斯)     概率性 | 后验概率   知识编码: 贝叶斯网络 + CPT
P6 (贝叶斯元推理)   融合性 | 似然比融合  融合: P2+P4+P5 → LR → 乘法
```

### 六种范式的递进路线

1. **P1-P3**：确定性推理的三种知识表示（OWL/Prolog/RDF），展示世界假设和推理方式的对比
2. **P4**：引入不确定性——从二元到连续，从"是/否"到"有多严重"
3. **P5**：引入概率论——从匹配度到后验概率，从启发式到数学保证
4. **P6**：多范式融合——将正交的增量信息统一为似然比，贝叶斯乘法融合

每一步都引入了前一步无法表达的信息维度，不是简单的重新编码。

## 九、已知限制与未来方向

### 条件独立假设

P2/P4/P5 共享同一份 diseases.csv 知识源，所以三个信号并非真正条件独立。这意味着乘法融合会过度放大一致性信号——三个引擎都"看到"同样的必要症状匹配，乘积会将这个同一信号放大三次。

在教学场景下这个妥协是可接受的——目标是展示融合框架本身。在生产系统中，可以通过以下方式缓解：
- 使用层次贝叶斯模型（Hierarchical Bayes）建模引擎间的相关性
- 从标注数据中经验校准 LR 值
- 引入去相关步骤

### P2 的 LR 为人工设定

P4 的 `exp(k*(c-0.5))` 和 P5 的 `posterior/prior` 都有理论依据，但 P2 的离散映射值（5.0/1.5/0.1）是专家设定的。在生产系统中，这些值应该从标注数据中经验校准。

### 充分症状未充分利用

diseases.csv 中的"充分症状"列（如"白细胞减少"对猫瘟）只有 P1/P2 使用。P4/P5 未建模充分症状。在更丰富的系统中，充分症状可以直接触发确诊。

## 十、总结

P6 的贝叶斯元推理不是简单的"加权平均"，而是利用了概率论的链式法则：

- 各引擎输出统一转为似然比，语义对齐
- 乘法融合有贝叶斯定理保证
- P1-P3 不再冗余运行，消除假一致性
- 每个引擎的证据贡献可量化（LR 值就是贡献度）

关键洞察：**多引擎融合的前提是各引擎提供正交的增量信息，而非冗余的重复判断。** 似然比是统一不同语义输出的标准工具——无论引擎输出的是二元判定、模糊匹配度还是后验概率，都可以转为 LR，然后以贝叶斯乘法融合。
