## 一、问题：医生在写字，不在看病

带宠物去看病的时候，你有没有注意过一个细节：医生一边问你问题，一边低头敲键盘。

"猫咪多大了？"——敲键盘。"食欲怎么样？"——敲键盘。"呕吐几次了？"——敲键盘。

一次门诊 15 分钟，可能有 8 分钟在写病历。医生真正看宠物、做检查、和你沟通的时间被压缩到很少。

这不是医生的问题，是系统的问题。传统宠物医院的电子病历全靠手敲，病历格式还不统一——今天这个医生这样写，明天换一个医生换个写法。等你想调历史病历时，发现半年前的记录像天书。

**为什么不能让医生只管说话，AI 来写病历？**

---

## 二、VetVoice 是什么

VetVoice 是我做的一个开源项目，一句话概括：

> **兽医声动，诊疗更高效。**

它是一个宠物医院 HIS 系统的智能电子病历插件，核心能力是把诊疗过程中的实时对话自动转换为结构化电子病历。

医生只需要正常和宠主交流，VetVoice 在后台完成四件事：

1. 实时语音识别——把对话转成文字
2. 说话人分离——自动区分哪些是医生说的，哪些是宠主说的
3. AI 辅助诊疗——根据对话内容自动整理病历
4. 一键导出——生成 PDF 病历 + 保存原始录音

## 三、技术架构

```
麦克风
   ↓
WebRTC APM（降噪）
   ↓
FunASR Streaming ASR（实时语音识别）
   ↓
Speaker Diarization（说话人分离）
   ↓
LLM Agent (MCP)（AI 辅助诊疗）
   ↓
结构化电子病历
```

每一层解决一个具体问题。

### 实时语音识别

诊室环境嘈杂——有猫叫、有狗叫、有空调声、有其他宠主说话。WebRTC APM 先做降噪处理，然后 FunASR 的流式 ASR 做实时识别。

选 FunASR 是因为它支持长语音连续识别，不需要说话停顿等模型追上来。医生和宠主的对话是自然流式的，识别也要跟上。

### 说话人分离

这一步很关键。如果对话混在一起不分人，AI 没法判断"呕吐三次"是医生说的还是宠主说的，病历就乱了。

VetVoice 用 wespeaker 做说话人 embedding，自动区分医生和宠主，在界面上用不同颜色显示对话。即使诊室里有三个人轮流说话，也能基本正确地分开。

### LLM Agent（MCP 协议）

这是 VetVoice 最有意思的部分。AI 不是简单地把对话转成文字就完事——它要理解诊疗流程，分阶段整理病历。

我把宠物诊疗拆成四个阶段：

- **问诊阶段**：主诉、病史、症状描述
- **开检查阶段**：血常规、生化、影像等检查项目
- **查看检查结果阶段**：异常指标分析
- **确诊治疗阶段**：诊断结论 + 用药 + 医嘱

Agent 基于 MCP 协议，每个阶段调用不同的工具完成病历结构的填充。LLM 的角色是理解和结构化——把"我家猫这两天老吐，吐了三四次，黄水"这样的口语，变成"呕吐，4次/2天，呕吐物为黄色液体"这样的病历语言。

## 四、为什么自己造轮子

市面上不是没有医疗语音转文字工具，但没有一个适合宠物医院。

人医的产品假设是：一个科室一个诊室，安静环境，标准术语。宠物医院的现实是：开放式诊台，猫狗齐叫，宠主用各种口语描述症状（"它不吃饭了""它蔫了""它拉稀了"），而且病历结构和人医完全不同。

更重要的是，宠物诊疗的 AI 辅助不是简单的语音转文字，而是需要理解宠物特有的疾病谱、用药规范和检查逻辑。通用的医疗 AI 在这里水土不服。

所以自己做。FunASR 解决语音识别，wespeaker 解决说话人分离，MCP Agent 解决病历结构化——每一层都选了当前最好的开源方案，组合起来就是一个完整的宠物诊疗语音助手。

## 五、和 OntologyOps 的关系

熟悉我的读者可能注意到了：VetVoice 和 OntologyOps 的宠物诊断示例（P1-P6）用的是同一个领域——宠物疾病诊断。

这不是巧合。

OntologyOps 探索的是**推理引擎**——给定症状，如何用 OWL/Prolog/模糊逻辑/贝叶斯等不同范式做诊断推理。VetVoice 探索的是**数据入口**——如何把诊疗对话自动变成结构化症状数据。

两者合在一起就是一条完整的链路：

```
VetVoice（对话 → 结构化数据）
   ↓
OntologyOps（结构化数据 → 推理诊断）
```

VetVoice 解决的是"数据从哪来"，OntologyOps 解决的是"数据怎么推理"。一个是感知，一个是认知。一个是语音 AI，一个是符号 AI。

## 六、使用方式

```bash
git clone https://github.com/georgewangchn/VetVoice.git
cd VetVoice
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

首次启动会自动下载 ASR 模型和说话人识别模型（支持断点续传）。然后在设置里填入 LLM 的 API 地址和密钥（兼容 OpenAI 格式），重启即可使用。

日常使用就三步：点"开始录音"→ 正常看诊 → 点"停止录音"然后"开始 Agent"，病历自动生成。

## 七、Roadmap

VetVoice 还在持续迭代：

- pyinstaller 打包 Windows/macOS/Linux 安装包，降低部署门槛
- 开放 HTTP/MCP 控制接口，让 HIS 系统可以直接调用

- 性能优化

## 八、写在最后

VetVoice 的出发点很简单：**让医生回归看病本身**。

技术在进步，但医疗的核心始终是人与人的沟通——医生和宠主之间的信任、观察、判断。这些不该被敲键盘的时间挤占。
