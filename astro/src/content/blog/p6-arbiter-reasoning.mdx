> 多范式推理实战营 · 项目 6/7  
> 副标题：当五种推理引擎意见不一致时，谁来裁决？

---

## 一、为什么需要融合

P1-P5 各自独立推理，各有优缺点：

| 引擎 | 优点 | 缺点 |
|------|------|------|
| P1 OWL | 双向推理，公理保证 | 不处理概率 |
| P2 Prolog | 递归推理，CWA 直接 | 过度排除 |
| P3 SPARQL | 前向链预计算 | 需要外部服务 |
| P4 模糊 | 处理灰度 | 无概率语义 |
| P5 贝叶斯 | 概率论保证 | 依赖先验质量 |

单一引擎的缺点，可以通过融合其他引擎来弥补。问题是：**当引擎们意见不一致时，听谁的？**

## 二、分层仲裁架构

```
第一层：确定性推理（P1 + P2 + P3）
  → 一致确诊/排除？直接采纳
  → 冲突？标记并进入下一层

第二层：模糊量化（P4）
  → 对疑似疾病做连续量化
  → 置信度 > 0.7 升级，< 0.3 降级

第三层：概率校准（P5）
  → 后验概率作为最终排序依据

仲裁器：综合三层裁决
  → 确定性一致 → 采纳
  → 确定性冲突 → 概率优先
  → 模糊与概率一致 → 加权融合（0.4 + 0.6）
  → 模糊与概率冲突 → 概率优先，标注冲突
```

### 为什么是这个分层顺序？

1. **确定性推理先行**：P1-P3 的"确诊"结论最强（公理保证），如果三个引擎都确诊，不需要后续层
2. **模糊量化居中**：P4 提供"匹配程度"信息，对确定性层无法裁决的灰色地带做量化
3. **概率校准殿后**：P5 的后验概率有严格数学基础，作为最终排序依据最可靠

### 权重设计

加权融合时：贝叶斯 0.6 + 模糊 0.4。概率权重更高的理由：
- 贝叶斯输出有概率论保证，模糊输出是启发式匹配度
- 贝叶斯利用先验知识（疾病发病率），模糊不利用
- 贝叶斯考虑负证据（未出现的症状），模糊忽略

## 三、冲突消解

### 场景 1：确定性层冲突

```
P1 (OWL): 猫瘟 → 确诊
P2 (Prolog): 猫瘟 → 排除（CWA 认为缺失症状=没有）
P3 (SPARQL): 猫瘟 → 确诊
```

仲裁：标记冲突，以 P5 贝叶斯后验概率为准。CWA 的"缺失=排除"过于激进，概率推理更平衡。

### 场景 2：模糊与概率冲突

```
P4 (模糊): 猫肠炎 置信度=0.70（高匹配度）
P5 (贝叶斯): 猫肠炎 P(D|S)=0.08（低后验概率）
```

仲裁：概率优先（0.08×0.6 + 0.70×0.4 = 0.33），标注冲突。差异 > 0.3 触发冲突标记。

## 四、推理结果示例

```
病例：猫，发热 + 呕吐 + 腹泻

1. 猫瘟    91.30%  █████████ [确诊（多引擎一致）]
   仲裁：确定性推理一致确诊（3/3引擎）
   P1: 1.00 [确诊]  P2: 1.00 [确诊]  P3: 1.00 [确诊]
   P4: 0.86 [高]    P5: 0.91 [高概率]

2. 猫肠炎   8.31%  [低置信度] ⚠️冲突
   仲裁：确定性层排除 vs 概率层有残留
   P1: 0.00 [排除]  P2: 0.00 [排除]  P3: 0.00 [排除]
   P4: 0.50 [中]    P5: 0.08 [低概率]
```

## 五、总结

P6 的分层仲裁不是简单的"少数服从多数"投票，而是利用了每种范式的优势：

- 确定性推理做**硬判断**（确诊/排除）
- 模糊推理做**软量化**（匹配程度）
- 概率推理做**最终校准**（后验排序）

当硬判断一致时，直接采纳；当硬判断冲突时，软量化 + 概率校准接管。
