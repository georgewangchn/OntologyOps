> 多范式推理实战营 · 项目 6/7  
> 副标题：当五种推理引擎意见不一致时，谁的证据该被采信？

---

## 一、为什么需要融合

P1-P5 各自独立推理，各有优缺点：

| 引擎 | 优点 | 缺点 |
|------|------|------|
| P1 OWL | 双向推理，公理保证 | 不处理概率 |
| P2 Prolog | 递归推理，CWA 直接 | 过度排除 |
| P3 SPARQL | 前向链预计算 | 需要外部服务 |
| P4 模糊 | 处理灰度 | 无概率语义 |
| P5 贝叶斯 | 概率论保证 | 依赖先验质量 |

单一引擎的缺点，可以通过融合其他引擎来弥补。问题是：**当引擎们意见不一致时，谁的证据该被采信？**

## 二、旧版方案的三个问题

最初的实现采用"分层仲裁"模式：P1+P2+P3 投票 → P4 模糊量化 → P5 概率校准 → 仲裁器加权(0.6/0.4)。从第一性原理审视后发现三个结构性缺陷：

### 问题 1：P1/P2/P3 知识源冗余

P1（OWL）、P2（Prolog）、P3（SPARQL）的知识源本质相同——同一份疾病-症状关系表的三种编码形式。三个引擎的置信度计算逻辑也完全相同（匹配数/必要症状数）。把它们放在"第一层"做投票，等价于一个人投三票，"一致性"几乎是必然的。

### 问题 2：P4 与 P1-P3 输入空间重叠

P4 模糊推理的三个输入中，`coverage`（覆盖率）的计算逻辑与 P1-P3 的置信度完全相同。P4 相对于 P1-P3 的真正增量仅为"症状严重度"（连续值），但旧版没有利用这个增量关系。

### 问题 3：权重 0.6/0.4 无理论依据

贝叶斯后验概率（P5）和模糊匹配度（P4）的语义不同。P5 的 0.8 = "后验概率 80%"，P4 的 0.8 = "模糊匹配度高"。把两者加权平均，就像把温度和湿度按 6:4 混合——单位不匹配。

## 三、贝叶斯元推理架构

```
病例输入 → ┬→ P2(Prolog) 确定性推理 → LR_struct
           ├→ P4(Mamdani) 模糊推理   → LR_fuzzy
           └→ P5(Naive Bayes) 概率推理 → LR_bayesian
                                            │
           Bayesian Meta-Reasoner ←────────┘
           P_final(D) ∝ P_prior(D) × LR_struct × LR_fuzzy × LR_bayesian
           归一化 → 最终后验概率分布
```

### 引擎选择：P1-P3 取其一

P1/P2/P3 共享同一知识源，只保留 P2（Prolog）。选择 P2 的理由：
- CWA（封闭世界假设）最适合诊断场景：未断言的症状 = 没有 → 可以直接排除
- P1 的 OWA 过于保守，P3 需要外部 Fuseki 服务

### 各引擎的增量信息

| 引擎 | 增量信息 | 语义 |
|------|---------|------|
| P2 | 疾病-症状必要性关系 + 排除关系 | 确定性：是/否 |
| P4 | 症状严重度（连续） | 模糊匹配度 |
| P5 | 先验概率 + 条件概率表 | 后验概率 |

三个引擎的增量信息是**正交的**——各自提供其他引擎没有的信息维度。

## 四、似然比转换

似然比（Likelihood Ratio, LR）是医学诊断中的标准工具。LR > 1 表示证据支持疾病，LR < 1 表示证据反对疾病。

### P2 确定性推理 → LR_struct

| P2 输出 | LR | 含义 |
|---------|-----|------|
| 确诊 | 5.0 | 强证据支持 |
| 疑似 | 1.5 | 弱证据支持 |
| 排除 | 0.1 | 强证据反对 |
| 未出现 | 1.0 | 无信息 |

### P4 模糊推理 → LR_fuzzy

使用指数映射：`LR = exp(k × (confidence - 0.5))`

- confidence = 0.5 → LR = 1.0（中性，不改变后验）
- confidence = 1.0 → LR ≈ 4.48（强支持）
- confidence = 0.0 → LR ≈ 0.22（强反对）

指数映射保证 LR 始终为正，且支持/反对区间对称。

### P5 贝叶斯推理 → LR_bayesian

`LR = posterior / prior`（后验相对先验的提升倍数）

- LR = 1.0 → 后验 = 先验（症状没提供额外信息）
- LR > 1.0 → 症状支持该疾病
- LR < 1.0 → 症状反对该疾病

## 五、贝叶斯乘法融合

```
P_final(D) ∝ P_prior(D) × LR_struct × LR_fuzzy × LR_bayesian
```

归一化后得到最终后验概率分布。这个公式有**贝叶斯定理的链式法则**保证——不是经验权重，而是概率论推导。

### 冲突检测

如果各引擎的 LR 方向不一致（一个大于 1 支持，一个小于 1 反对），且最大 LR / 最小 LR 超过 5.0，标记冲突。

## 六、推理结果示例

```
病例：猫，发热 + 呕吐 + 腹泻

1. 猫瘟    91.30%  █████████ [高概率]
   融合：P_prior(0.0500) × LR_P2(5.0) × LR_P4(3.2) × LR_P5(18.2)
   P2: conf=1.00 [确诊] LR=5.00
   P4: conf=0.86 [高]   LR=3.20
   P5: conf=0.91 [高概率] LR=18.20

2. 猫肠炎   5.20%  [低概率]
   融合：P_prior(0.0300) × LR_P2(0.1) × LR_P4(1.8) × LR_P5(0.8)
   P2: conf=0.00 [排除] LR=0.10
   P4: conf=0.62 [中]   LR=1.80
   P5: conf=0.02 [低概率] LR=0.80
```

## 七、总结

P6 的贝叶斯元推理不是简单的"加权平均"，而是利用了概率论的链式法则：

- 各引擎输出统一转为似然比，语义对齐
- 乘法融合有贝叶斯定理保证
- P1-P3 不再冗余运行，消除假一致性
- 每个引擎的证据贡献可量化（LR 值就是贡献度）

关键洞察：**多引擎融合的前提是各引擎提供正交的增量信息，而非冗余的重复判断。**
